⚠️ WebRTC Camera Access Requirements

🔒 HTTPS Required for Camera/Microphone Access

Your browser requires a secure connection (HTTPS) to access camera and microphone for streaming.

Why is this happening?

Modern browsers block camera/microphone access on non-secure (HTTP) connections for security reasons. This is a browser security feature that cannot be bypassed.

✅ Solutions:

  1. Option 1: Use localhost (Development)
    Access the app at: http://localhost:3000
    Browsers allow WebRTC on localhost even without HTTPS.
  2. Option 2: Set up HTTPS (Production)
    Configure your server with an SSL certificate (Let's Encrypt is free).
  3. Option 3: Use ngrok or similar tunneling service
    ngrok http 3000
    This provides a temporary HTTPS URL for testing.
  4. Option 4: Chrome Flag (Testing Only)
    Chrome: Navigate to chrome://flags/#unsafely-treat-insecure-origin-as-secure
    Add your URL: http://:3000
    ⚠️ This is insecure and should only be used for testing.
