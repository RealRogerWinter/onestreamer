🛡️ XSS Protection Test Suite

Test various XSS payloads to ensure chat is protected

Test Payloads:

Try copying these into the chat to verify they don't execute:

1. Basic Script Tag:
<script>alert('XSS')</script>
2. Image Tag with onerror:
<img src=x onerror=alert('XSS')>
3. SVG with onload:
<svg onload=alert('XSS')>
4. Iframe injection:
<iframe src=javascript:alert('XSS')></iframe>
5. HTML formatting (should not render):
<b>bold</b> <i>italic</i> <font color="red">red text</font>
6. JavaScript URL:
<a href="javascript:alert('XSS')">Click me</a>
7. Data URL with script:
<a href="data:text/html,<script>alert('XSS')</script>">Click</a>
8. Event handler injection:
<div onclick="alert('XSS')">Click me</div>
9. Style with expression:
<div style="background:url('javascript:alert(1)')">test</div>
10. Mixed with emoji (should only render emoji):
<script>alert('XSS')</script> :kek: <img src=x onerror=alert('XSS')>

✅ Expected Behavior:

  • All HTML tags should be displayed as plain text
  • No JavaScript should execute
  • No pop-ups or alerts should appear
  • Emoji codes like :kek: should still render as images
  • Regular URLs should still be clickable

🔒 Security Measures Implemented:

  • Client-side: HTML escaping before rendering
  • Client-side: DOMPurify sanitization with strict config
  • Server-side: HTML entity encoding
  • HTTP Headers: Content Security Policy
  • HTTP Headers: X-XSS-Protection
  • Allowed HTML: Only <img> for emojis and <a> for links
